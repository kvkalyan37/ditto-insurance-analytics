# .gitignore Explanation

This `.gitignore` file excludes unwanted files from version control.

## What Gets Ignored:

### ✅ Data Files
- `*.csv` - Scraped data files (generated, not source code)
- `data/` - Data directories
- Database files

### ✅ Python Artifacts
- `__pycache__/` - Python bytecode cache
- `*.pyc` - Compiled Python files
- `venv/`, `env/` - Virtual environments
- `*.egg-info/` - Package metadata

### ✅ Logs
- `*.log` - All log files
- `logs/` - Log directories
- Service-specific logs

### ✅ IDE Files
- `.vscode/` - VSCode settings (except shared configs)
- `.idea/` - IntelliJ/PyCharm settings
- Editor-specific files

### ✅ OS Files
- `.DS_Store` - macOS
- `Thumbs.db` - Windows
- System files

### ✅ Secrets & Configs
- `.env` - Environment variables
- `kubeconfig` - Kubernetes configs
- `*.key`, `*.pem` - Certificates

### ✅ Build Artifacts
- `dist/`, `build/` - Build outputs
- `node_modules/` - Node dependencies
- Compiled files

## What Gets Kept:

### ✅ Source Code
- `*.py` - Python source files
- `*.js`, `*.html`, `*.css` - Frontend files
- `Dockerfile` - Container definitions

### ✅ Configuration
- `requirements*.txt` - Python dependencies
- `.github/workflows/*.yml` - CI/CD workflows
- `helm/` - Helm charts
- `nginx.conf` - Nginx configuration

### ✅ Documentation
- `*.md` - Markdown files
- `README.md` - Project documentation

## Why Ignore CSV Files?

The `ditto_insurance_data.csv` file is:
- Generated by the scraper
- Updated every 30 minutes
- Large and changes frequently
- Not source code

It's better to:
- Generate it during CI/CD
- Include it in Docker images
- Not track it in Git

## Customization

To include specific files that match ignore patterns, use:
```
!filename.csv  # Force include this file
```

To exclude additional patterns, add them to `.gitignore`.
